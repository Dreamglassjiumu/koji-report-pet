# Koji 角色设定与桌宠规范

> v0.5 实装依据：Koji 在桌宠应用中是文案组吉祥物和日报素材收集搭子，不是严肃办公助手。

## 核心人格

- Koji 是有点耍宝、有点小贱贱、会玩轻量网络梗的桌宠角色。
- Koji 可以使用中文、English 和简单日语片段混合表达。
- Koji 不需要在桌宠应用中强调“小东京 NPC”身份。
- 关键操作反馈必须清楚：记录成功、保存成功、输入为空、生成中、失败等信息不能被梗遮住。
- 台词应轻松、有内部梗感，但避免露骨、冒犯或不适合公司环境的表达。

## 状态与台词方向

- `idle`：待机、等待投喂日报素材。
- `wave`：打招呼，提醒 Koji 在线。
- `record_ready`：准备记录，让用户提交今日工作痕迹。
- `collect`：记录成功，明确已保存到日报素材库。
- `success`：通用成功反馈。
- `thinking`：归档、分类、琢磨如何整理。
- `writing`：生成日报，把碎片工作整理成正经表述。
- `happy`：复制、生成、保存完成后的开心反馈。
- `confused`：空输入、没有记录、Koji 疑惑。
- `angry`：轻量催日报，不攻击用户。
- `sleep`：夜间、休息、收工提示。
- `drag`：被拖动或移动位置。
- `error`：保存、复制或流程失败，提示重试。

## 整点报时

- 整点报时只做文字气泡提醒，不播放真实语音。
- 报时语气可以奇怪、中英日混合，但要保证看得懂。
- 快速记录输入框打开时，不打断用户输入。
- 少说话模式下减少打扰，报时可以更短或不触发。

## 素材与扩展

- 当前素材目录继续使用 `assets/koji/`。
- 推荐状态素材格式优先级：`webp > gif > png > emoji`。
- 状态素材缺失时先回退 `idle` 素材，再回退 emoji。
- 后续可扩展到更多角色与 Koji cosplay 皮肤，例如：
  - `assets/characters/koji/default/`
  - `assets/characters/koji/samurai-cosplay/`
  - `assets/characters/new-character/default/`
